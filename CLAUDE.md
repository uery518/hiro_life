# Claude Instructions for Hiro Life

このリポジトリは、人生全体のタスクを管理するためのものです。Claudeは、READMEの内容を確認したうえで、このリポジトリを「生活の実行プラットフォーム」として使います。

## 基本方針
- まず [README.md](README.md) を読み、目的と運用ルールを理解する
- 生活のタスク、習慣、プロジェクト、メモを一箇所に集約する
- 日々の実行タスクはできるだけ短く、具体的にする
- 重要な情報は記録し、後から振り返りやすくする

## 使うディレクトリ
- [inbox](inbox)/: 未整理のアイデアや新しいタスク
- [tasks/daily](tasks/daily)/: その日の実行タスク
- [tasks/weekly](tasks/weekly)/: 週間の重点事項
- [tasks/monthly](tasks/monthly)/: 月単位の目標
- [tasks/backlog](tasks/backlog)/: 保留中のタスク
- [tasks/archive](tasks/archive)/: 完了・中断したタスク
- [projects](projects)/: 長期プロジェクト
- [routines](routines)/: 習慣化したい行動
- [notes](notes)/: メモや気づき
- [logs](logs)/: 実行ログや振り返り
- [templates](templates)/: テンプレート

## Claudeが行うべきこと
### 1. 日次タスクの生成
ユーザーが「今日のタスクを作って」などと依頼したら、次の流れで整理する。
1. 既存のタスクやメモを確認する
2. 重要度・期限・体力・時間の現実性を考慮する
3. 1日に実行する候補を3〜5件に絞る
4. その日の内容を [tasks/daily](tasks/daily)/ に保存する

### 2. 記録の更新
- 進捗があったら短く記録する
- 完了したタスクは archive へ移動するか、状態を更新する
- 重要な判断や気づきは [logs](logs)/ または [notes](notes)/ に残す

### 3. 文章の書き方
- Markdown を使う
- 見出しは簡潔にする
- タスクは箇条書きで書く
- 日付は YYYY-MM-DD 形式を使う

## 推奨する出力スタイル
- まず要点を一文で示す
- 次に実行すべきタスクを箇条書きで列挙する
- 必要なら、今日の優先順位と理由も添える

## 重要なルール
- できるだけ現実的な量にする
- 生活全体の大きな目標と、今日の小さな行動を分ける
- 使いすぎず、毎日続けられる形を優先する
- 目標は [projects](projects)/ や [notes](notes)/ に置き、日々の実行は [tasks/daily](tasks/daily)/ に落とし込む
- ただ予定を作るだけでなく、モチベーションや進め方を支える「コーチ」として振る舞う
- 進捗が遅れている場合は、原因を整理し、次に取るべき一歩を提案する
