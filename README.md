# Hiro Life Task Management

このリポジトリは、人生全体のタスクを一元管理するための基盤です。
仕事・学習・健康・家事・習慣・長期プロジェクト・メモなど、生活のあらゆることをここに集約し、Claudeに自動で毎日の優先タスクを作らせることを目的としています。

## 目的
- すべてのタスクを一箇所に集約する
- 期限・重要度・状況に応じて毎日計画を作る
- 長期目標と日常タスクを分けて管理する
- Claudeが読みやすく、継続的に使いやすい構造にする

## Claudeに期待する運用
Claudeはこのリポジトリを参照して、次のように動くと理想的です。

1. 今日の文脈と、今週・今月の優先事項を確認する
2. 未完了タスク・期限・習慣・プロジェクト状態を読み取る
3. 1日に実行可能な重要タスクを3〜5件に絞って提案する
4. その日の成果を tasks/daily/YYYY-MM-DD.md にまとめる
5. 重要な進捗や気づきを logs/ や notes/ に記録する

## 推奨ディレクトリ構成

```text
.
├── README.md
├── CLAUDE.md
├── inbox/
│   └── ideas.md
├── tasks/
│   ├── daily/
│   ├── weekly/
│   ├── monthly/
│   ├── backlog/
│   └── archive/
├── projects/
├── routines/
├── notes/
├── logs/
└── templates/
```

## 各ディレクトリの役割
- inbox/: すぐに処理したいアイデアや未整理のタスク
- tasks/daily/: その日の実行タスク
- tasks/weekly/: 1週間の重点事項
- tasks/monthly/: 月単位の目標と計画
- tasks/backlog/: 完了まで保留しているタスク
- tasks/archive/: 完了・中断したタスクの履歴
- projects/: 長期プロジェクトごとのメモや計画
- routines/: 習慣化したい行動のルール
- notes/: 生活・学習・気づきなどの一般メモ
- logs/: 実行ログや振り返りの記録
- templates/: 日次・週次・プロジェクト用のテンプレート

## 運用ルール
- ファイルは基本的に Markdown で管理する
- 日付は YYYY-MM-DD 形式を使う
- 1タスクは1行程度で短く書く
- 毎日の計画は現実的な量にする
- 重要な決定や振り返りは logs/ に残す

## 目標の置き場所
- 長期的な目標や理想の状態は [projects](projects)/ または [notes](notes)/ に置く
- 週単位の重点事項は [tasks/weekly](tasks/weekly)/ に置く
- 月単位の目標は [tasks/monthly](tasks/monthly)/ に置く
- その日の行動に落とし込むときは [tasks/daily](tasks/daily)/ を使う

## コーチングとしての使い方
- Claudeには「ただタスクを列挙する」だけでなく、優先順位や現実的な進め方を考える役割を持たせる
- 目標に対して、今の行動が足りているか、無理がないかを確認する
- 進捗が止まっているときは、原因を整理し、次の一歩を提案する
- 毎日のタスクは、達成しやすい小さなステップに分ける

## 使い始めるときのおすすめ
- まずは tasks/daily/ と tasks/weekly/ を使い始める
- 生活全体の大きなテーマは projects/ で管理する
- Claudeに「今日のタスクを生成して」と依頼したら、README とディレクトリ構成を参照させる

この構成により、Hiroの人生を管理するための「単一の作業台」として使いやすくなります。
